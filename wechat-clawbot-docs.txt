### 一、架构原理

```
微信用户 → iLink Bot（一对一绑定）→ getUpdates 长轮询 → OpenClaw Gateway → rc-wechat-agent → AI 回复 → 微信用户
```

**核心机制**：iLink 的 bot token 是一对一绑定的，每个微信用户需要独立扫码生成自己的 bot token。openclaw-weixin 插件为每个账号启动独立的 `getUpdates` 长轮询。

### 关键文件

| 文件 | 说明 |
|------|------|
| `/root/.openclaw/openclaw-weixin/accounts.json` | 账号索引（数组，记录所有 accountId） |
| `/root/.openclaw/openclaw-weixin/accounts/{botId}.json` | 账号凭证（token、baseUrl、userId） |
| `/root/.openclaw/openclaw-weixin/accounts/{botId}.sync.json` | getUpdates 同步状态 |
| `/root/.openclaw/openclaw-weixin/accounts/{botId}.context-tokens.json` | 会话 context token |
| `/root/.openclaw/openclaw.json` | Gateway 配置（channelConfigUpdatedAt 触发热重载） |

### 账号凭证格式

```json
{
  "token": "{botId}:{secret}",
  "savedAt": "2026-05-08T14:48:00.000Z",
  "baseUrl": "https://ilinkai.weixin.qq.com",
  "userId": "{wechatUserId}@im.wechat"
}
```

### 二、新用户接入流程（手动）

1. 调用 iLink API 生成二维码
2. 用户微信扫码确认
3. 轮询扫码状态，获取 bot_token / ilink_bot_id / ilink_user_id
4. 写入 `accounts/{botId}.json`
5. 注册到 `accounts.json` 索引
6. 更新 `openclaw.json` 的 `channelConfigUpdatedAt` 触发 Gateway 热重载
7. （可选）在 RC 数据库 `wechat_bindings` 表中记录绑定关系

### 三、自动化接入 API

#### 3.1 生成二维码

```
POST /api/wechat/bindbot/qrcode
```

**响应**：
```json

  "qrcode_url": "https://liteapp.weixin.qq.com/q/7GiQu1?qrcode=...&bot_type=3",

```

#### 3.2 轮询扫码状态

```
GET /api/wechat/bindbot/status?session_key={session_key}
```

**响应（等待中）**：
```json

```

**响应（已扫码）**：
```json
{ "status": "scaned" }
```

**响应（需要验证码）**：
```json
{ "status": "need_verifycode" }
```

**响应（确认成功）**：
```json
{
  "status": "confirmed",
  "bot_id": "410860c3bb94@im.bot",
  "user_id": "o9cq804lF_5T-QiBhox4oFnrGpZ4@im.wechat",
  "message": "绑定成功，微信助手已激活"
}
```

#### 3.3 提交验证码

```
POST /api/wechat/bindbot/verify
Content-Type: application/json

{ "session_key": "uuid", "code": "1234" }
```

### 七、注意事项

1. **一对一绑定**：每个微信用户对应一个独立的 bot token 和 getUpdates 轮询
2. **热重载**：注册新账号后通过更新 `channelConfigUpdatedAt` 触发 Gateway 热重载，无需重启
3. **避免重启循环**：不要用 `systemctl restart openclaw-gateway`，用热重载方式
4. **Token 安全**：账号凭证文件权限设为 600（仅 owner 可读写）
5. **会话过期**：扫码会话 5 分钟过期，过期后需重新生成二维码
6. **验证码**：部分用户扫码后需要输入手机微信显示的数字验证码

### 技术实现细节

#### 后端实现方式
- **不使用 OpenClaw Gateway 代理**（Gateway 未暴露 HTTP API）
- **直接调用 iLink API**：
  - 二维码生成: `POST https://ilinkai.weixin.qq.com/ilink/bot/get_bot_qrcode?bot_type=3`
  - 状态轮询: `GET https://ilinkai.weixin.qq.com/ilink/bot/get_qrcode_status?qrcode={token}`

#### 请求头和认证
- **Content-Type**: `application/json`
- **Bot Token**: 从本地账号文件读取并包含在 `local_token_list` 参数中

#### 前端访问链接
- **公开页面**: `https://repaceclaw.com/wechat-qr` (无需登录)
- **管理页面**: `https://repaceclaw.com/wechat-clawbot`
- **纯HTML版本**: `https://repaceclaw.com/wechat-qr.html`

### 使用流程

1. **调用二维码接口** → 获取 `qrcode` token 和二维码图片URL
2. **显示二维码** → 使用 `original_qrcode_url` 或通过 `/api/wechat-clawbot/qrcode/image?id=xxx` 访问
3. **轮询状态** → 每2秒调用状态接口，直到 `status` 变为 `"confirmed"`
4. **获取凭据** → 登录成功后获得 `bot_token` 用于后续IM通信

### 故障排除

#### 常见问题
- **"Channel Status: Not configured"**: 前端显示问题，实际功能正常
- **API 404错误**: 确保后端直接调用 iLink API，而非代理到 Gateway
- **权限问题**: 确保账号文件权限为 644 (`chmod 644 *.json`)

### 多账号上下文隔离

默认情况下，私聊可能共用同一会话桶。**多个微信号同时登录**时，建议按「账号 + 渠道 + 对端」隔离：

```bash
openclaw config set session.dmScope per-account-channel-peer
```

### 自定义 BotAgent（可选）

每条出站请求会带一个自我声明的 `bot_agent` 字段——类似 HTTP `User-Agent`——用于后台日志归因和监控聚合。**默认值为 `OpenClaw`**。

在 `openclaw.json` 中加一行即可：

```json
{
  "channels": {
    "openclaw-weixin": {
      "botAgent": "MyBot/1.2.0"
    }
  }
}
```

**格式规范**（UA 风格）：
- 一个或多个 `Name/Version` token，空格分隔
- 每个 token 可选地跟一个 ` (comment)`
- 仅允许 ASCII 字符；总长 ≤ 256 字节
- 不合规的 token 在清洗时静默丢弃；如果最终为空，回退到 `OpenClaw`

### 后端 API 协议

本插件通过 HTTP JSON API 与后端网关通信。所有接口均为 `POST`，请求和响应均为 JSON。

#### 通用请求头

| Header | 说明 |
|--------|------|
| `Content-Type` | `application/json` |
| `AuthorizationType` | 固定值 `ilink_bot_token` |
| `Authorization` | `Bearer <token>`（登录后获取） |
| `X-WECHAT-UIN` | 随机 uint32 的 base64 编码 |

#### 接口列表

| 接口 | 路径 | 说明 |
|------|------|------|
| getUpdates | `getupdates` | 长轮询获取新消息 |
| sendMessage | `sendmessage` | 发送消息（文本/图片/视频/文件） |
| getUploadUrl | `getuploadurl` | 获取 CDN 上传预签名 URL |
| getConfig | `getconfig` | 获取账号配置（typing ticket 等） |
| sendTyping | `sendtyping` | 发送/取消输入状态指示 |

#### getUpdates 详情

长轮询接口。服务端在有新消息或超时后返回。

**请求体**：
```json
{ "get_updates_buf": "" }
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `get_updates_buf` | `string` | 上次响应返回的同步游标，首次请求传空字符串 |

**响应体**：
```json
{
  "ret": 0,
  "msgs": [...],
  "get_updates_buf": "<新游标>",
  "longpolling_timeout_ms": 35000
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `ret` | `number` | 返回码，`0` = 成功 |
| `errcode` | `number?` | 错误码（如 `-14` = 会话超时） |
| `errmsg` | `string?` | 错误描述 |
| `msgs` | `WeixinMessage[]` | 消息列表 |
| `get_updates_buf` | `string` | 新的同步游标 |
| `longpolling_timeout_ms` | `number?` | 服务端建议的下次长轮询超时（ms） |

#### sendMessage 详情

**请求体**：
```json
{
  "msg": {
    "to_user_id": "<目标用户 ID>",
    "context_token": "<会话上下文令牌>",
    "item_list": [
      { "type": 1, "text_item": { "text": "你好" } }
    ]
  }
}
```

#### getUploadUrl 详情

获取 CDN 上传预签名参数。上传文件前需先调用此接口获取 `upload_param` 和 `thumb_upload_param`。

**请求体**：
```json
{
  "filekey": "<文件标识>",
  "media_type": 1,
  "to_user_id": "<目标用户 ID>",
  "rawsize": 12345,
  "rawfilemd5": "<明文 MD5>",
  "filesize": 12352,
  "thumb_rawsize": 1024,
  "thumb_rawfilemd5": "<缩略图明文 MD5>",
  "thumb_filesize": 1040
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `media_type` | `number` | `1` = IMAGE, `2` = VIDEO, `3` = FILE |
| `rawsize` | `number` | 原文件明文大小 |
| `rawfilemd5` | `string` | 原文件明文 MD5 |
| `filesize` | `number` | AES-128-ECB 加密后的密文大小 |
| `thumb_rawsize` | `number?` | 缩略图明文大小（IMAGE/VIDEO 时必填） |
| `thumb_rawfilemd5` | `string?` | 缩略图明文 MD5（IMAGE/VIDEO 时必填） |
| `thumb_filesize` | `number?` | 缩略图密文大小（IMAGE/VIDEO 时必填） |

#### getConfig 详情

**请求体**：
```json
{
  "ilink_user_id": "<用户 ID>",
  "context_token": "<可选，会话上下文令牌>"
}
```

**响应体**：
```json
{
  "ret": 0,
  "typing_ticket": "<base64 编码的 typing ticket>"
}
```

#### sendTyping 详情

**请求体**：
```json
{
  "ilink_user_id": "<用户 ID>",
  "typing_ticket": "<从 getConfig 获取>",
  "status": 1
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | `number` | `1` = 正在输入，`2` = 取消输入 |

### 消息结构

#### WeixinMessage

| 字段 | 类型 | 说明 |
|------|------|------|
| `seq` | `number?` | 消息序列号 |
| `message_id` | `number?` | 消息唯一 ID |
| `from_user_id` | `string?` | 发送者 ID |
| `to_user_id` | `string?` | 接收者 ID |
| `create_time_ms` | `number?` | 创建时间戳（ms） |
| `session_id` | `string?` | 会话 ID |
| `message_type` | `number?` | `1` = USER, `2` = BOT |
| `message_state` | `number?` | `0` = NEW, `1` = GENERATING, `2` = FINISH |
| `item_list` | `MessageItem[]?` | 消息内容列表 |
| `context_token` | `string?` | 会话上下文令牌，回复时需回传 |

#### MessageItem

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | `number` | `1` TEXT, `2` IMAGE, `3` VOICE, `4` FILE, `5` VIDEO |
| `text_item` | `{ text: string }?` | 文本内容 |
| `image_item` | `ImageItem?` | 图片（含 CDN 引用和 AES 密钥） |
| `voice_item` | `VoiceItem?` | 语音（SILK 编码） |
| `file_item` | `FileItem?` | 文件附件 |
| `video_item` | `VideoItem?` | 视频 |
| `ref_msg` | `RefMessage?` | 引用消息 |

#### CDN 媒体引用 (CDNMedia)

所有媒体类型（图片/语音/文件/视频）通过 CDN 传输，使用 AES-128-ECB 加密：

| 字段 | 类型 | 说明 |
|------|------|------|
| `encrypt_query_param` | `string?` | CDN 下载/上传的加密参数 |
| `aes_key` | `string?` | base64 编码的 AES-128 密钥 |

### CDN 上传流程

1. 计算文件明文大小、MD5，以及 AES-128-ECB 加密后的密文大小
2. 如需缩略图（图片/视频），同样计算缩略图的明文和密文参数
3. 调用 `getUploadUrl` 获取 `upload_param`（和 `thumb_upload_param`）
4. 使用 AES-128-ECB 加密文件内容，PUT 上传到 CDN URL
5. 缩略图同理加密并上传
6. 使用返回的 `encrypt_query_param` 构造 `CDNMedia` 引用，放入 `MessageItem` 发送

### 核心逻辑

```typescript
// 1. 组件挂载时：
//    - 建立 SSE 连接（/api/wechat-clawbot/events）
//    - 获取连接状态（/api/wechat-clawbot/status）
//    - 每 15 秒轮询一次状态

// 2. 生成二维码：
//    - POST /api/wechat-clawbot/qrcode
//    - 获取 qrcode_url 和 qrcode token
//    - 开始轮询扫码状态

// 3. 轮询扫码状态：
//    - POST /api/wechat-clawbot/qrcode/status
//    - 每 2 秒轮询一次
//    - 最多 120 次（约 4 分钟）
//    - confirmed 时刷新连接状态

// 4. SSE 事件处理：
//    - state: 更新 wsState
//    - connected: 刷新状态
//    - disconnected: 更新状态
//    - gateway_event: 添加到事件日志
```
